Статус резервного копирования

Сервер выполняет инкрементное резервное копирование почти каждую ночь. Резервные копии можно хранить в любом сервисе, совместимом с Amazon S3, или выбрать другие варианты.

Настройка

Резервные копии хранятся на локальном диске этого сервера. Вы несёте ответственность за периодическое копирование (через SFTP — FTP поверх SSH) файлов резервных копий из в надёжное место. Эти файлы зашифрованы, поэтому их безопасно хранить где угодно.

Отдельно скопируйте пароль шифрования из в безопасное и надёжное место. Этот файл потребуется для расшифровки резервных копий.

Резервные копии синхронизируются на удалённый сервер с помощью rsync по SSH. Локальные копии остаются в . Эти файлы зашифрованы, поэтому их безопасно хранить где угодно.

Отдельно скопируйте пароль шифрования из в безопасное и надёжное место. Этот файл потребуется для расшифровки резервных копий.

Имя хоста у вашего провайдера rsync, например da2327.rsync.net. При необходимости можно указать через двоеточие нестандартный SSH-порт провайдера, например u215843.your-storagebox.de:23.
Скопируйте приведённый выше публичный SSH-ключ и вставьте его в файл ~/.ssh/authorized_keys пользователя на целевом сервере резервного копирования. Это позволит настроить защищённую аутентификацию без пароля между вашим сервером Mail-in-a-Box и сервером резервных копий.

Резервные копии хранятся в корзине, совместимой с S3. У вас уже должна быть учётная запись в AWS или другом сервисе S3-совместимого хостинга.

Резервные копии хранятся в корзине Backblaze B2. У вас уже должна быть учётная запись Backblaze.

Это минимальный срок хранения данных резервных копий в днях. Сервер делает инкрементные резервные копии в большинстве ночей, что требует сохранения предыдущих копий вплоть до последней полной, поэтому данные часто хранятся дольше, чем указано здесь. Полные резервные копии делаются периодически, когда размер инкрементных данных превышает порог.
